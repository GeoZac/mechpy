Intro to python and gotchas

To get started, choose the python distribution you want. I would recommend anaconda, which has most of the scientific packages that are needed, but more importantly, comes with a package manager called conda, which will help us update and install new packages and manage environments if we want to use multiple versions of python. miniconda is a great open-source lightweight distribution, but I would recommend anaconda":"http://continuum.io/downloads#34" which will install many scientific packages.

Once the install is complete, lets set up our virtual environments with conda. This will allow you to have an isolated environment for different projects. I typically default with the most recent version and create a 2.7 environment.

to check what environments are install type conda info -e

To install a full version of python 2.7 with many packages, type conda create -n py27 python=2.7 anaconda

In linux activate different environments by 

source activate py27
source deactivate

and in windows 

activate py27
deactivate

It is easy to install new packages with conda or pip 

conda install numpy
conda install pandas
conda install pywin32
pip install python-pptx
pip install pyinstaller
pip install moviepy

Updating 1 packages or all is just as easy 

conda update matplotlib

# or update all
conda update --all

Since python is a generic computing language, numerical computation was not initially a very high priority, and so there are a few quirks of the python language that may cause some confusion for non-programmers or matlab users. The first is the difference of python version 2.7 and the current suggested version python 3+ (as of 2016). Many legacy packages exist for python 2.7, so it is handy to use, but it is recommended to develop in the current release.

a handy tool to ensure so errors go unnoticed is to import the future module. This will take care of the print function syntax change print('hi') and integer division / 

from __future__ import print_function, division

The second is variable assignment. Since every variable in python is an object, we have to copy variables, not just use =. for example, to copy a variable 

# CORRECT
from copy import copy
x = 5
y=copy(x)

# WRONG
y=x

Importing Python Modules

When using python, the very first code you will need is to import modules. An example of importing the essential scientific computing libraries with some settings is shown here. 

# setup
import numpy as np
import sympy as sp
import scipy
from pprint import pprint
sp.init_printing(use_latex='mathjax')

import matplotlib.pyplot as plt
plt.rcParams['figure.figsize'] = (12, 8)  # (width, height)
plt.rcParams['font.size'] = 14
plt.rcParams['legend.fontsize'] = 16
from matplotlib import patches

get_ipython().magic('matplotlib')  # seperate window
get_ipython().magic('matplotlib inline') # inline plotting

# or for inline plotting, in an  ipython console, use the magic commands
%matplotlib inline

# to revert to window plotting
%matplotlib
#or
%matplotlib qt
